    # Project Reactor Operators

    Documenation of webflux operators with simple examples and their usage

    # Basics

    • Publisher : which produces data
      • Flux : publisher that can emit one or more elements
      • Mono : publisher that can emit one or zer element
    • Subscriber: Which listens to the data produced by publisher

    Eventhough most of the operators listed below are for Flux, they can be still used with Mono.

    # all

    # Usage

    Can be used when we want to test wether all elements emitted by publisher passes a given predicate function. params: Predicate result: Boolean

    # Example

    // prints 1,2,3
Flux.just(1,2,3)
    .subscribe(d -> System.out.println(d));

    we can add predicate to test.

    Flux.just(1,2,3)
    .all(integer -> integer > 0)
    .subscribe(d -> System.out.println(d)); 
    // prints true as elements published are greater than zero

Flux.just(1,2,3,-1)
    .all(integer -> integer > 0)
    .subscribe(d -> System.out.println(d)); 
    // prints false , since all elements didn't pass the predicate

    # any

    # Usage

    Can be used when we want to test wether any element emitted by publisher passes a given predicate function. params: Predicate result: Boolean

    # Example

    Flux.just(1,2,3)
    .subscribe(d -> System.out.println(d));
    // prints 1,2,3

    we can add predicate to test.

    Flux.just(1,2,3)
    .any(integer -> integer > 2)
    .subscribe(d -> System.out.println(d)); 
    // prints true as element published has element 3 which is greater than 2

Flux.just(1,2,3,-1)
    .any(integer -> integer > 3)
    .subscribe(d -> System.out.println(d)); 
    // prints false , since all elements are less than 3
